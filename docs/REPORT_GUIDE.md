# JCNIRS レポートガイド — 評価指標と図の読み方

本書は研究レポート「**量子アニーリング (Amplify QUBO) による特徴量選択の有用性**」の
作成支援のため、`run_all.py` が出力する **評価指標** と **図 (figures/)** の意味・
読み方・そこから導ける主張を詳しくまとめたものです。

- 指標の実装: [metrics.py](../metrics.py)（`compute_metrics` / `METRIC_INFO`）
- 図の実装  : [visualization.py](../visualization.py)
- 値はすべて **5-fold 交差検証の OOF（Out-Of-Fold）予測** に基づき、**元スケール
  （含水率 [%]）** で計算されます（学習は log1p 空間でも、評価は逆変換後）。

## 1. 研究の狙いと指標・図の対応

| 研究目標 | 主に使う指標 | 主に使う図 |
|----------|--------------|------------|
| (1) 含水率予測の精度向上 | RMSE / R² / RPD / RPIQ | 予測 vs 実測、残差、指標ヒートマップ、戦略×指標比較 |
| (2) 量子アニーリング特徴量選択の有用性 | RMSE / R²（同特徴量数での比較）、特徴量数 | 選択波長、Jaccard 重なり、関連度-冗長度、特徴量数 vs 精度 |

「有用性」は次の3点で論じられます:
- **精度**: 同じ特徴量数で QUBO が古典 FS 以上の精度を出すか（戦略×指標比較／特徴量数 vs 精度）
- **選択の質**: QUBO が **高関連・低冗長** な波長を選べているか（関連度-冗長度図）
- **選択の独自性**: QUBO の選ぶ波長が古典法とどれだけ異なるか（Jaccard／選択波長図）

## 2. 評価指標の詳細

すべて OOF 予測 ŷ と実測 y（含水率 [%]）から計算します。CV 結果表では `CV-RMSE`,
`CV-R2` のように `CV-` 接頭辞が付きます。

### RMSE — 二乗平均平方根誤差
- **式**: √(平均((y − ŷ)²))
- **範囲 / 単位**: 0〜∞、含水率と同じ [%]。**低いほど良い**
- **意味**: 誤差を二乗平均して平方根を取った値。大きな誤差を強く罰するため、
  外れの予測があると敏感に悪化する。**本コンペの主要指標**であり、本パイプラインの
  モデル選択・アンサンブル選択の基準。

### MAE — 平均絶対誤差
- **式**: 平均(|y − ŷ|)
- **範囲 / 単位**: 0〜∞、[%]。**低いほど良い**
- **意味**: 典型的な誤差の大きさ。外れ値の影響が RMSE より小さい。
  **RMSE ≫ MAE** のときは一部に大外しがあることを示唆する（両者の併記が有効）。

### R² — 決定係数
- **式**: 1 − Σ(y−ŷ)² / Σ(y−ȳ)²
- **範囲**: −∞〜1（1 が完全、0 は「常に平均で予測」と同等、負は平均以下）。**高いほど良い**
- **意味**: 目的変数の分散のうちモデルが説明できた割合。モデルの説明力の総合指標。

### RPD — 残差予測偏差比（Ratio of Performance to Deviation）
- **式**: std(y) / RMSE
- **範囲**: 0〜∞。**高いほど良い**
- **意味**: NIR 分光分析で最も標準的な実用性指標。実測のばらつき（標準偏差）に対し
  誤差がどれだけ小さいか。**目安**: <1.5 不十分 / 1.5–2.0 粗い判別可 / 2.0–2.5 良好 /
  2.5–3.0 非常に良好 / >3.0 優秀。
- 本データは std≈49.5。RMSE が 25 なら RPD≈2.0。

### RPIQ — 四分位範囲性能比（Ratio of Performance to InterQuartile distance）
- **式**: IQR(y) / RMSE（IQR = 第3四分位 − 第1四分位）
- **範囲**: 0〜∞。**高いほど良い**
- **意味**: RPD の頑健版。標準偏差の代わりに IQR を使うため、**分布の歪みや外れ値に
  強い**。本データの含水率は右に裾を引く（mean 50 > median 29）ため、RPD より RPIQ の
  方が実態に即しやすい。**目安**: >2.0 良好 / >2.5 非常に良好。IQR≈54。

### Bias（ME）— 平均誤差
- **式**: 平均(ŷ − y)
- **範囲**: −∞〜∞（**0 が理想**）
- **意味**: 系統的な偏り。正なら全体的に過大予測、負なら過小予測。0 から離れるほど
  キャリブレーションがずれている。RMSE が小さくても Bias が大きいと系統誤差が残る。

### MAPE — 平均絶対パーセント誤差
- **式**: 平均(|(y − ŷ) / y|) × 100 [%]
- **範囲**: 0〜∞ [%]。**低いほど良い**
- **意味**: スケール非依存の相対誤差。**注意**: 含水率が小さいサンプル（本データに
  <5% が 64 件）で分母が小さく値が極端に膨らむため、**補助指標**として参照する
  （RMSE/R²/RPD を主、MAPE を従とする）。

### MaxError — 最大絶対誤差
- **式**: max(|y − ŷ|)
- **範囲 / 単位**: 0〜∞、[%]。**低いほど良い**
- **意味**: 最も外した 1 サンプルの誤差。**最悪ケース**の把握に使う。

## 3. 図の詳細

図は `figures/` に出力されます。ファイル名と対応を示します。図中テキストは
日本語フォント非依存にするため英語表記です（意味は本書で補足）。

### 3.1 EDA（データ概観）

#### `eda_target_distribution.png` — 含水率の分布
- **内容**: 左=生の含水率ヒストグラム＋統計量、中=箱ひげ図、右=log1p 変換後ヒストグラム。
- **読み方**: 生分布は右に強く裾を引き（mean≫median, 外れ大）回帰が難しい形。log1p 後は
  分布が対称に近づく。
- **主張**: 「右裾の重い分布だから log1p 変換で学習を安定化した」という**前処理の妥当性**
  の根拠図。

#### `eda_spectra_overview.png` — スペクトル概観
- **内容**: 上=平均スペクトル±1std とサンプルスペクトル、下=各波長と含水率の |相関| 曲線。
- **読み方**: 下段で相関が高い波数帯が含水率の情報を多く持つ領域。水の吸収帯
  （およそ 5200・6900 cm⁻¹ 付近など）にピークが立つことが多い。
- **主張**: 含水率予測に効く波長帯の存在＝特徴量選択に意味があることの裏付け。

### 3.2 戦略ごと（`<strategy>_` 接頭辞）

`<strategy>` は none / filter / wrapper / embedded / amplify。

#### `<strategy>_heatmap_metrics.png` — モデル×前処理の指標ヒートマップ
- **内容**: RMSE / R² / RPD について、行=モデル・列=前処理のヒートマップ。
  **緑=良い、赤=悪い**（指標の向きに応じて配色を反転済み）。
- **読み方**: どのモデル×前処理が強いか一目で分かる。行（モデル）・列（前処理）の
  傾向（例: SG 微分系が効く、勾配ブースティングが安定 等）を読む。

#### `<strategy>_barplot_model_metrics.png` — モデル別ベストの指標バー
- **内容**: 各モデルの「ベスト前処理」での RMSE / R² / RPD を横棒で比較。
- **読み方**: モデル間の優劣ランキング。深層モデル（CNN/DeepSpectra/Transformer/AE系/GAN）と
  古典モデル（PLS/Ridge/LightGBM 等）の比較に使う。

#### `<strategy>_pred_vs_actual.png` — 予測 vs 実測
- **内容**: 左=ベスト単体モデル、右=ベストアンサンブル。横=実測、縦=OOF 予測、赤破線=1:1。
  右上に RMSE/R²/RPD を注記。
- **読み方**: 点が 1:1 線に密集するほど高精度。低含水率域での圧縮や高含水率域での
  飽和（系統的なずれ）を視認できる。
- **主張**: モデルの予測品質を直感的に示す代表図（レポートの本文向き）。

#### `<strategy>_residuals.png` — 残差プロット
- **内容**: 左=残差(予測−実測) vs 予測値、右=残差ヒストグラム。
- **読み方**: 左が水平な帯状に均一なら良好。傾き/扇形（分散が予測値で変わる=
  不均一分散）や偏り（0 から外れる=Bias）を検出。右が 0 中心の正規形なら理想。

### 3.3 戦略横断（レポートの核）

#### `compare_strategy_metrics.png` — 戦略×指標
- **内容**: RMSE / R² / RPD / RPIQ について、5 戦略（各戦略のベスト pipeline）の値を棒で比較。
- **読み方**: 量子アニーリング（amplify）が他戦略と並ぶ／上回るかを指標ごとに確認。
- **主張**: **「QUBO が古典 FS と同等以上の精度を達成」**を示す中心図。

#### `compare_features_vs_accuracy.png` — 特徴量数 vs 精度
- **内容**: 横=選択特徴量数、縦=ベスト RMSE（左）/ R²（右）。各点に戦略名を注記。
- **読み方**: 左下（少特徴で低 RMSE）が理想。none（全 1555 波長）に対し、200 波長の
  各 FS がどこまで迫れるか。
- **主張**: **「QUBO は少ない特徴量で高精度＝効率的」**という有用性の根拠。

#### `compare_runtime.png` — 戦略別の実行時間
- **内容**: 戦略ごとの所要時間（特徴量選択＋全モデル CV）。
- **読み方**: amplify は QUBO 求解時間（既定 5000ms）を含む。精度向上に対する計算コストの
  位置づけを議論できる。

#### `compare_feature_count.png` — 選択特徴量数
- **内容**: 戦略別に実際に選ばれた特徴量数の棒。
- **読み方**: 各 FS が設定どおり 200 波長を選べているか、none が全波長かを確認。

#### `compare_selected_wavelengths.png` — 選択波長の可視化 ★重要
- **内容**: 平均スペクトル上に、各戦略が選んだ波長を縦線（赤帯）で重ね描き（戦略ごとに段）。
- **読み方**: filter（MI）は相関の高い帯に密集＝**隣接波長が固まりやすい（冗長）**。
  amplify（QUBO/mRMR）は冗長を避けるため**より分散して広い帯に散る**傾向が見える。
- **主張**: **「QUBO は冗長を避けて多様な波長を選ぶ」**ことの視覚的証拠。

#### `compare_feature_overlap.png` — Jaccard 重なり ★重要
- **内容**: 戦略間で選択特徴量集合の Jaccard 類似度（|共通| / |和|）をヒートマップ化。
- **読み方**: 1 に近いほど似た特徴を選ぶ。amplify と他法の値が低いほど **QUBO の選択が独自**。
- **主張**: QUBO が古典法と異なる解に到達していることの定量的証拠。

#### `compare_qubo_diagnostics.png` — 関連度 vs 冗長度 ★最重要
- **内容**: 各戦略の選択特徴について、左=平均関連度 |corr(波長, 含水率)|（高いほど良い）、
  中=平均冗長度（選択特徴間の平均 |corr|、低いほど良い）、右=関連度-冗長度平面の散布。
- **読み方**: mRMR の理想は「右上かつ低冗長（=右図で上かつ左）」。amplify が **高関連を保ちつつ
  低冗長**を実現していれば QUBO 定式化が効いている証拠。filter は関連度は高いが冗長度も
  高くなりがち。
- **主張**: **「QUBO は mRMR 目的（最大関連・最小冗長）を実際に達成している」**——
  量子アニーリング定式化の有用性を最も直接的に裏付ける図。
- **注**: ここでの関連度・冗長度は計算が軽い **Pearson |相関|** に基づく代理指標。
  実際の QUBO の関連度は相互情報量（MI）を用いている（[feature_selection_quantum.py](../feature_selection_quantum.py)）。

## 4. レポートでの主張の組み立て方

**精度向上の物語**:
1. `eda_target_distribution` で課題（右裾分布）→ log1p 変換の導入を説明。
2. `<strategy>_heatmap_metrics` / `barplot_model_metrics` で best モデル・前処理を特定。
3. `<strategy>_pred_vs_actual` / `residuals` でベスト構成の予測品質を提示。
4. RPD/RPIQ を NIR 分野の基準（RPD>2.0 等）と照らして実用性を述べる。

**量子アニーリングの有用性の物語**:
1. `compare_strategy_metrics`：amplify が古典 FS と同等以上の精度。
2. `compare_features_vs_accuracy`：少ない特徴量で高精度（効率）。
3. `compare_qubo_diagnostics`：高関連・低冗長を達成（mRMR 目的の実現）。
4. `compare_selected_wavelengths` / `compare_feature_overlap`：選択波長が独自で分散的。
5. `compare_runtime`：精度・選択品質の利得に対する計算コストの位置づけ。

## 5. 注意点・限界

- **amplify 戦略の前提**: Amplify SDK 未インストール、または `AMPLIFY_TOKEN` 未設定だと
  amplify はスキップされます。その場合、戦略横断図は **実行できた戦略のみ**で描画されます。
  量子アニーリングの比較を行うには `.env` にトークンを設定してください。
- **OOF 指標**: 表・図の指標は 5-fold CV の OOF 値であり、コンペ提出（テスト）スコアとは
  別物です（提出は `submissions_<strategy>/sub_BEST.csv`）。
- **MAPE**: 低含水率サンプルで過大評価されうるため主要指標にはしないでください。
- **関連度-冗長度図の代理指標**: 3.3 の注のとおり、図は相関ベース、QUBO 本体は MI ベース。
  厳密一致ではなく傾向の可視化として解釈してください。
- **乱数依存**: 深層モデルや KFold はシード固定（`random_state=42`）ですが、環境
  (CPU/GPU・ライブラリ版) により微小な差が出ることがあります。
